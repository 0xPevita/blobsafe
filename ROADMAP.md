# BlobSafe Roadmap

> Building the default private storage layer for Web3 teams.

---

## ✅ Phase 1 - Foundation (Current)

- [x] Project scaffolding with Next.js 14
- [x] Shelby Protocol SDK integration
- [x] Aptos Wallet (Petra) connection
- [x] Client-side AES-256-GCM encryption
- [x] Drag-and-drop file upload
- [x] Virtual folder structure via blob naming
- [x] File list with Shelby account query
- [x] SHA-256 integrity verification per upload
- [x] S3 Gateway compatibility

---

## 🔨 Phase 2 - Core Features (Next)

- [ ] File download with client-side decryption
- [ ] File preview (images, PDFs, text)
- [ ] Folder creation and management UI
- [ ] File deletion and expiration management
- [ ] Upload progress with real-time percentage
- [ ] Drag-and-drop between folders

---

## 🌐 Phase 3 - Team & Access Control

- [ ] Multi-user workspace support
- [ ] On-chain access control via Aptos Move contract
- [ ] Grant / revoke access to specific files or folders
- [ ] Time-limited share links (enforced on-chain)
- [ ] Audit log - all read/write events on Aptos
- [ ] Multi-sig for destructive operations (delete, transfer)

---

## 🚀 Phase 4 - Power Features

- [ ] Full S3 Gateway UI (bucket browser)
- [ ] DuckDB integration query Parquet files directly
- [ ] Storage analytics dashboard (usage, bandwidth, cost)
- [ ] Verifiable storage receipt NFT
- [ ] CLI tool for developers
- [ ] REST API for programmatic access

---

## 🏁 Phase 5 - Mainnet Launch

- [ ] Security audit of encryption module
- [ ] Mainnet deployment on Shelby
- [ ] Token integration (ShelbyUSD payments)
- [ ] Public launch + documentation site
- [ ] Community governance for protocol parameters

---

*Roadmap is subject to change based on Shelby Protocol development and community feedback.*
